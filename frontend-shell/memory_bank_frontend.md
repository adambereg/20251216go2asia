# 1. Общий контур UI Go2Asia

Go2Asia v1 — это модульная система UI, построенная вокруг отдельных вертикалей:

- Atlas Asia — страны, города, места, гайды, темы
- Pulse Asia — календарь событий
- Blog Asia — статьи, категории, темы
- Russian Friendly — партнёры, отзывы, ваучеры
- Rielt Asia — каталог недвижимости
- Space Asia — социальная сеть, сообщества, посты
- Guru Asia — карта объектов и сервисов
- Connect Asia — кошелёк, уровни, рефералы, аналитика
- Quest Asia — квесты, лидеры, прогресс

UI построен на принципах:

- PWA-shell
- Сегментированных роут-групп: (public), (authenticated), (auth)
- Единой дизайн-системы
- Повторяющихся карточных паттернов
- Типовых шаблонов страниц (list → details → related content)

# 2. Универсальные UX-паттерны, обязательные к переносу

### 2.1. Карточный фреймворк

Все модули используют общий паттерн карточек:

- PlaceCard  
- EventCard  
- ArticleCard  
- PartnerCard  
- ListingCard  
- PostCard  
- QuestCard  

Общие свойства карточек:

- Фото всегда в рамке
- Над карточкой — мета-информация (тема, категория, страна)
- Под фото — заголовок, краткое описание
- Справа/снизу — CTA (Подробнее / Купить / Пройти / Присоединиться)

### 2.2. Структура страниц

Любой модуль следует схеме:

**1. Overview Page**  
→ Заголовок  
→ Hero-секция  
→ Категории / фильтры  
→ Ленты карточек  

**2. Details Page**  
→ Hero + breadcrumbs  
→ Основной контент  
→ Карта (если есть геопозиция)  
→ Блок «похожие» / «тематические»

### 2.3. Фильтры и сортировки

Модули используют:

- текстовые фильтры
- фильтры по категориям
- фильтры по геопозиции / стране / городу
- календарные фильтры (Pulse)
- фильтры по цене (Rielt, RF)

### 2.4. Поведение UI

- Skeleton-загрузчики являются частью UX
- Все списки используют карточные гриды или вертикальные списки
- Все детали используют structured layout: info → media → related → actions

# 3. Архитектура модулей UI (сжатая модель)

```

```

components/[module]/utils/

### 3.3. Типовая структура

```

  components/
    [Х]-Card.tsx
    [Х]-List.tsx
    FilterBar.tsx
  utils/
  types.ts
  mockData.ts (только референс)
```

# 4. Дизайн-система — обязательный набор знаний

### 4.1. Основы

- Tailwind config расширен в `tailwind.config.full.js`
- Брендовые токены — в `design-tokens.json`
- Компоненты используют единую spacing-scale
- Цветовая система:
  - primary: #1677FF
  - secondary: #52C41A
  - accent: #FF4D4F

### 4.2. Базовые UI-компоненты

- Button
- Badge
- Card.Base
- Input / Select
- SectionHeader
- Chip / Tag
- Avatar
- Skeleton

### 4.3. Визуальные принципы

- Радиусы элементов гладкие (md-lg)
- Фото — всегда с рамкой
- Hero — крупный текст + фото
- Пропорции карточек: вертикальные (3:4 или 4:5)

# 6. MIGRATION-MODE (как агент переносит UI)

### 6.1. Приоритеты переноса

1. UX-структуры страниц  
2. Карточные паттерны  
3. Фильтры + списки  
4. Дизайн-токены  
5. Layout-композиции

### 6.2. Не переносим

- Legacy импорты  
- mockData.ts  
- Старая аутентификация  
- Прямые API-коллы  
- Старые хуки React Query  

### 6.3. Обязательные адаптации

- Новая структура путей  
- Новая архитектура модулей  
- Новая дизайн-система  
- Новая система данных / агентов  
- Новые API контракты  

---

# 7. GOLDEN RULES

1. Старый UI — источник знаний.  
2. Важна идея, а не конкретный JSX.  
3. Карточки — фундамент.  
4. Герои сохраняют стиль.  
5. Истина — docs/ui/*.md  
6. Всегда адаптировать.  
7. Модули независимы.  

---

# 8. Формула преобразования страницы

```
1. Определи тип страницы.
2. Изучи docs/ui/[module]_ui.md.
3. Извлеки паттерны.
4. Создай новый skeleton.
5. Перенеси разметку → адаптируй.
6. Подключи токены и UI-компоненты.
7. Удали моки → подключи реальные данные.
8. Проверь UX-паттерны.
```
