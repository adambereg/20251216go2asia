# ADR-003: Отказ от расширенной структуры (skills/, prompts/, guidelines/, memory/, evaluation/) на этапе MVP

## Статус
Accepted

## Дата
2025-01-03

## Контекст
В мультиагентных системах нередко создаются дополнительные уровни:
- skills (атомарные способности),
- prompts (варианты промтов),
- guidelines (правила разработки),
- memory (долговременные факты),
- evaluation (тестирование агентов).

Однако на текущем этапе развития Go2Asia:
- структура должна быть минимальной,
- важна скорость разработки,
- система ещё не требует строгой автоматизации и версионирования промтов.

## Решение
Принято решение **не создавать** указанные директории на этапе MVP.  
Базовая структура ограничена:
- roles/
- workflows/
- decisions/

## Обоснование
### Плюсы
- минимизация шума,
- упрощение навигации,
- снижение когнитивной нагрузки для агентов,
- отсутствие дублирования.

### Минусы
- позже может потребоваться расширение,
- skills могут пригодиться при автоматизации.

### Альтернативы
1. Создать сразу все структуры  
— отказано: перегруз и документационная инерция.  
2. Создать только prompts/  
— отказано: роли уже содержат системные промты.

## Последствия
- документация остаётся компактной,
- минимальный набор файлов: roles/, workflows/, adr,
- дальнейшее расширение возможно при необходимости.
