# UI Kit — базовые принципы интерфейса Go2Asia

UI Kit определяет визуальные и интерактивные основы фронтенда Go2Asia.  
Он используется всеми модулями: Atlas, Pulse, Blog, Guru, Space, Quest, RF, Rielt и др.

---

# 1. Философия дизайна Go2Asia

- **Минимализм + информационная плотность**
- **Дружелюбный интерфейс для путешественников**
- **Повторяемые паттерны во всех модулях**
- **Мобильный приоритет (Mobile First)**
- **Быстрая ориентация в контенте (карточки, превью, списки)**

---

# 2. Основные компоненты интерфейса

## 2.1. Заголовки
- H1 — крупный, жирный, используется в hero-секциях.
- H2 — разделы страниц внутри модулей.
- H3 — подпункты, фильтры, боковые блоки.

Шрифт по умолчанию: **Inter variable**.

---

## 2.2. Текстовые блоки
- Body — основное текстовое поле.
- Caption — вспомогательные подписи.
- Badge — маленькие маркеры категорий.

---

## 2.3. Кнопки (Buttons)
Типы:
- Primary — акцентная, зелёная (brand).
- Secondary — контурная.
- Tertiary — текстовая.

Состояния:
- hover,
- active,
- disabled,
- loading (spinner).

---

## 2.4. Карточки контента
Используются в:
- Atlas (страны, города, места),
- Pulse (события),
- Blog (статьи),
- Guru (заведения),
- RF-партнёры (каталог),
- Rielt (объявления).

Единые правила карточек описаны в `components/cards.md`.

---

## 2.5. Поля ввода
Типы:
- текстовое поле,
- выпадающий список,
- селектор даты/времени,
- поиск (с иконкой лупы).

Детали в `components/forms.md`.

---

## 2.6. Иконки
Набор: Lucide Icons.  
Размеры:
- 16px,
- 20px,
- 24px (по умолчанию).

---

# 3. Цветовая система

Определяется через design tokens (см. `design_tokens.md`).

Основные цвета:
- зелёный brand,
- графитовый нейтральный,
- белый фон,
- акцентные оттенки для статусов.

---

# 4. Структура страниц

Каждая страница:
1. Hero-блок или заголовок страницы,
2. Фильтры / поисковая строка,
3. Контент в виде карточек, списков или сеток,
4. Подвал (footer) — единый для всех модулей.

---

# 5. Навигация

Основная навигация выполнена на уровне PWA shell:

- Bottom Navigation (мобильный),
- Sidebar Navigation (десктопный),
- Header с поиском и быстрыми ссылками.

Детали — в `layouts/pwa_shell.md`.

---

# 6. Стандарты для модулей

Каждый модуль использует единые UI-правила:

- одинаковые размеры карточек,
- одинаковая структура hero,
- одинаковые кнопки фильтров,
- одинаковая типографика.

Детализировано в `layouts/module_layouts.md`.

---

# 7. Адаптивность

Основной принцип: **Mobile-first, Desktop-refined**.

Детали — в `mobile_vs_desktop.md`.

---

# 8. Гайдлайны для Cursor и AI-разработчиков

- UI должен следовать design tokens.
- Карточки должны соответствовать компонентам из design/components/.
- Layout только из layouts/.
- Контрастность — WCAG AA.
- Ни одного самостоятельного цвета, только токены.
