# UI Review Checklist (MOCK DATA)

**Контекст:** UI-first моковый контент для Pulse / Atlas / Blog.

**Переключатель источника данных:**
- `NEXT_PUBLIC_DATA_SOURCE=mock|api`
- По умолчанию: `mock`
- В режиме `mock` страницы показывают бейдж **MOCK DATA** в шапке.

## 20 пунктов для визуальной проверки
1. **Навигация/роутинг:** переходы между `/pulse` → `/pulse/[id]`, `/blog` → `/blog/[slug]`, `/atlas/places/[id]` работают без 404.
2. **Бейджи источника данных:** в режиме `mock` на страницах видно **MOCK DATA**, в режиме `api` — не видно (или видно явное пустое состояние).
3. **Лоадеры:** лоадер не прыгает по высоте (layout shift) и не ломает шапку/hero.
4. **Пустые состояния:** при отсутствии данных (особенно в `api` режиме) текст понятный, CTA/ссылка назад есть.
5. **Ошибки:** ошибки сети/5xx (если включён `api`) отображаются без "мерцания" и без бесконечных спиннеров.
6. **Карточки в списках:** одинаковые высоты карточек/гридов, нет визуальных "ступенек".
7. **Переполнение текста:** длинные заголовки/описания обрезаются корректно (line-clamp), без вылезания.
8. **Кнопки/CTA:** все CTA кликабельны, имеют hover/focus состояния.
9. **Фокус и клавиатура:** можно табом пройти по интерактивным элементам, фокус виден.
10. **Мобилка (375px):** hero/шапка не обрезается, меню/чипы скроллятся горизонтально без артефактов.
11. **Планшет (768px):** сетки переходят в 2 колонки там, где ожидается; нет лишних пустых полей.
12. **Десктоп (1280px+):** максимальная ширина контента адекватна, нет "растяжки" текста на весь экран.
13. **Изображения:** при отсутствии cover/hero — есть аккуратный fallback; изображения не "ломаются" (broken image).
14. **Контраст:** текст поверх изображений читаем (градиенты/оверлеи достаточные).
15. **Локализация RU:** даты/строки на русском, форматирование дат выглядит естественно.
16. **Иконки:** иконки выровнены по baseline, не скачут по размеру.
17. **Бейджи/чипы:** много тегов не ломает layout; происходит перенос/обрезка.
18. **Деталка Pulse:** блоки (описание, место, дата/время) не расползаются при длинных значениях.
19. **Деталка Atlas place:** ключевые поля (тип, координаты, категории) читаемы и не дублируются.
20. **Деталка Blog:** длинный контент читаем (межстрочный, отступы, списки), сайдбары не перекрывают контент.
