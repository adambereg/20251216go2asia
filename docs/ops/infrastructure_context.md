# Infrastructure Context (IMMUTABLE)

Этот документ фиксирует уже настроенное окружение проекта Go2Asia.
Считается ИСТОЧНИКОМ ПРАВДЫ для всех архитектурных и технических решений.

⚠️ ВАЖНО:
- Запрещено предлагать альтернативные провайдеры и деплой-пайплайны
- Запрещено повторно настраивать описанные сервисы
- Любые изменения допускаются ТОЛЬКО по прямому запросу владельца проекта

---

## Frontend

- Хостинг и деплой: **Netlify**
- CI/CD: Netlify build from GitHub (будет включён после готовности фронтенда)
- Предполагаемый фреймворк: Next.js / React (уточняется в архитектуре)
- CDN: Cloudflare (см. ниже)

---

## Edge / Infra

- **Cloudflare**
  - DNS: настроен
  - Workers: созданы и используются
  - R2 Object Storage: создан и доступен
- Используется как edge-слой, CDN и gateway

---

## Backend

- Деплой: **GitHub Actions**
- Архитектура: микросервисы (детализация в архитектурных документах)
- Среда исполнения: определяется в backend docs (Cloudflare Workers / Node / etc.)

---

## Database

- Провайдер: **Neon**
- Тип: PostgreSQL
- Статус:
  - База создана
  - Таблицы существуют
  - В базе есть демо-контент
- Запрещено:
  - пересоздавать схему
  - “предлагать другую БД”
- Разрешено:
  - добавлять миграции поверх существующей схемы

---

## Auth

- Провайдер: **Clerk**
- Статус: настроен и используется
- Запрещено:
  - внедрять альтернативные auth-системы
  - дублировать логику аутентификации

---

## Summary (TL;DR)

- Netlify — frontend deploy
- Cloudflare — DNS / CDN / Workers / R2
- Neon — PostgreSQL
- Clerk — Auth
- GitHub Actions — backend deploy

Любые отклонения от этого контекста считаются ошибкой архитектуры.
