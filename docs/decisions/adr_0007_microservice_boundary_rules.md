# ADR 0007: Правила выделения границ микросервисов

## Статус
Принято

## Дата решения
2025-12-10

## Контекст
Go2Asia содержит больше 20 микросервисов. Нужно определить правила их выделения.

## Решение
Микросервис выделяется, если:

1. имеет собственную модель данных,
2. использует уникальные бизнес-правила,
3. может масштабироваться отдельно,
4. является точкой интеграции с внешними системами,
5. может работать независимо от других.

## Последствия
- чёткие boundaries,
- меньше монолитных эффектов,
- предсказуемый рост системы.

## Альтернативы
Создавать сервисы «как получится» → ведёт к хаосу.
