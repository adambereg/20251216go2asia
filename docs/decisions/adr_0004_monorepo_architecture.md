# ADR 0004: Выбор монорепозитория как основной структуры проекта

## Статус
Принято

## Дата решения
2025-12-10

## Контекст
Go2Asia состоит из десятков фронтенд-модулей, десятков микросервисов и общей дизайн-системы.  
Управлять множеством отдельных репозиториев неэффективно.

## Решение
Использовать **монорепозиторий с pnpm + Turborepo**.

## Причины
- единый доступ к общим библиотекам,
- единый CI/CD,
- кэширование сборок,
- удобное разделение "капсул" для Cursor,
- упрощение тестирования.

## Последствия
Повышается скорость разработки, вводится больше порядка, но возрастает ответственность за структуру.

## Альтернативы
- polyrepo → хаос и дублирование кода.
