# RF Service — Roadmap

## Этап 1 — MVP каталога Russian Friendly

- Бизнес-партнёры + одна основная точка.
- Базовые поля:
  - имя, описание, категория, страна/город, контакты.
- Статусы партнёров:
  - `pending_review`, `active`.
- Публичный каталог:
  - фильтры по стране/городу/категории,
  - простая сортировка.
- Базовая связь с Atlas (через `atlas_place_id`).

## Этап 2 — Мульти-точки и PRO-онбординг

- Несколько точек у одного партнёра.
- Роли `partner_owner` / `partner_manager`.
- Полноценный онбординг через PRO:
  - `onboarding_pro_user_id`,
  - списки “моих” партнёров у PRO.

## Этап 3 — Интеграция с Voucher / Premium-логикой

- Поле `default_voucher_ids`, `default_premium_voucher_ids`.
- Конфигурация экономики Премиум-ваучеров на уровне партнёра:
  - `premium_business_reward_g2a`,
  - `premium_pro_reward_g2a`.
- Internal API `/internal/partners/{id}/basic` для Token/Voucher/Quest.

## Этап 4 — Углублённая интеграция с Points / Quest / Space

- Конфигурируемые правила кэшбэка в Points.
- Связка квестов с партнёрами (квесты в партнёрских местах).
- Публикация избранного UGC-контента о партнёрах
  (чтобы Blog/Space могли показывать “лучшее из соцсети”).

## Этап 5 — Аналитика и AI-подбор партнёров

- Расширенная статистика:
  - топ партнёров по странам/городам,
  - активность PRO по онбордингу.
- Подбор партнёров на основе профиля пользователя:
  - рекомендации “куда сходить рядом” с учётом предпочтений и истории.
- Автоматические подсказки PRO:
  - какие бизнесы стоит привлечь в партнёрку в конкретном районе.
