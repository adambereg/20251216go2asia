# Pulse Service — Обзор

## Назначение

Pulse Service — микросервис календаря событий для экосистемы Go2Asia (модуль **Pulse Asia**).  
Он отвечает за хранение и выдачу информации о мероприятиях: фестивалях, экскурсиях, встречах, концертах, турах и других активностях, происходящих в странах ЮВА. Сервис одновременно выполняет роль:

- **Афиши “здесь и сейчас”** — список актуальных предстоящих событий;
- **Исторического архива** — база прошедших мероприятий, отражающая “ритм жизни” региона.

Для пользователя это “что происходит вокруг”, для платформы — фундамент событийного слоя, который используют Atlas, Guru, Space, Blog, Quest и Connect.

## Основные функции

- Хранение событий:
  - название, краткое и полное описание;
  - категорий/тип (концерт, тур, экскурсия, митап, фестиваль и т.п.);
  - даты и времени проведения (начало, окончание, таймзона);
  - статуса (черновик, опубликовано, отменено, завершено);
  - связанного места `place_id` из Atlas или RF-партнёра;
  - организатора (PRO-пользователь, бизнес-партнёр RF, редакция Go2Asia).

- Два режима использования:
  - **Афиша**: предстоящие события с фильтрами по датам (сегодня, завтра, выходные, диапазон), городу, категории, ценам.
  - **Архив**: прошедшие события для аналитики, контент-планирования и статей (Blog).

- Интеграция с пользователями и партнёрами:
  - создание событий редакцией;
  - создание и ведение событий PRO-организаторами;
  - события бизнес-партнёров RF (вечеринки, дегустации, спецпредложения);
  - предложения событий от обычных пользователей с модерацией.

- API для:
  - получения списков событий с фильтрами;
  - показа событий рядом (через Guru);
  - детальных карточек события.

- Интеграция с другими сервисами:
  - **Atlas** — привязка к месту через `place_id`, гео-фильтры;
  - **Guru** — выдача событий “рядом с тобой”;
  - **Space** — отметки “иду/интересно”, чаты по событиям;
  - **Blog (Media)** — репортажи и анонсы;
  - **Quest** — использование событий как шагов квеста;
  - **Connect / Tokenomics** — учёт активности PRO-организаторов, награды Points/NFT.

## Роли пользователей

- **Гость** — видит публичную афишу и детали событий.
- **Зарегистрированный пользователь** — может подписываться на события (Фаза 2), сохранять избранные (Фаза 2).
- **PRO-пользователь** — может создавать и редактировать свои события, видеть базовую аналитику.
- **Бизнес-партнёр RF** — создавать события, привязанные к заведению.
- **Редактор/Админ** — модерировать, публиковать, размещать официальные события Go2Asia.

## Связь с фронтенд-модулем Pulse Asia

Модуль **Pulse Asia** использует Pulse Service как back-end:

- разные виды календаря: список, сетка, календарь по дням/неделям/месяцам;
- фильтры по городам/странам/категориям;
- детальная страница события с местом, организатором, ссылками в Space и Blog;
- виджеты событий для встраивания в Atlas, Guru и другие модули.
