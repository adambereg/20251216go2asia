# Media Service — Обзор

## Назначение

Media Service — бэкенд для **Blog Asia**: медиаплатформы экосистемы Go2Asia.  
По сути это **headless CMS**, которая обеспечивает создание, хранение и выдачу статей:

- редакционного контента (редакция Go2Asia),
- материалов от PRO и бизнес-партнёров,
- отобранных UGC-постов из соцсети Space Asia.

Blog Asia — “официальное медиа” экосистемы, витрина лучших текстов. Space — живая соцсеть. Media Service — мост между ними.

## Основные функции

- **Публикация и хранение статей**:
  - заголовок, подзаголовок, текст, медиа (обложка, галерея, встраиваемое видео),
  - SEO-метаданные: `slug`, мета-описания, OG-поля,
  - статусы: `draft`, `pending_review`, `scheduled`, `published`, `rejected`, `archived`.  

- **Управление контентом**:
  - черновики и отложенные публикации (расписание),
  - версии/правки статьи и возможность отката,
  - модерация материалов от PRO/партнёров и импортируемых UGC.

- **Рубрикатор**:
  - категории (рубрики блога),
  - теги,
  - тематические хабы (страницы “Темы”).

- **Связь с сущностями экосистемы**:
  - привязка статьи к:
    - стране/городу/месту (Atlas),
    - событию (Pulse),
    - бизнес-партнёру/заведению (RF),
    - квесту (Quest).  

  Это позволяет на странице города/места/события показывать подборку релевантных статей.

- **Интеграция с Space (соцсеть)**:
  - импорт лучших пользовательских постов как статей (с согласия автора),
  - создание анонса в Space при публикации статьи (пост-ссылка/тред обсуждения).  

- **Интеграция с Connect / Tokenomics**:
  - генерация событий вроде `content_featured`, `content_popular`
  - начисление Points/NFT за популярные статьи (через Token/Points/NFT сервисы).  

## Роли и пользователи

- **Читатель (guest/user)** — читает статьи.
- **PRO** — может предлагать материалы (черновики/на модерацию).
- **Партнёр (RF)** — публикует партнёрские статьи/новости о своём бизнесе (через модерацию).
- **Редактор** — редактирует, утверждает, планирует публикации.
- **Админ** — полный контроль, управление категориями/тегами и системными настройками.

## Связь с фронтенд-модулем Blog Asia

Модуль **Blog Asia** использует Media Service как API:

- главные ленты:
  - “Главная” — смесь свежих и избранных статей,
  - лента по рубрике (категории),
  - лента по теме/тегу,
  - подборки по городам/странам;
- просмотр статьи:
  - контент, автор, привязки к местам/событиям,
  - блок “Ещё по теме”, “Ещё в этом городе”;
- виджеты:
  - блоки “Последние статьи по этому городу” на страницах Atlas,
  - “Материалы по событию” на страницах Pulse.
