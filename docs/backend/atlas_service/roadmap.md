# Atlas Service — Roadmap

## Статус к MVP

На момент MVP (первые итерации Go2Asia): :contentReference[oaicite:14]{index=14}  

- реализован Atlas как:
  - базовый справочник стран и городов;
  - первый набор мест для ключевых локаций (Фукуок, Бангкок и др.);
- API:
  - чтение стран/городов/мест;
  - базовые фильтры;
- интеграция:
  - Guru — получает места по радиусу;
  - Pulse и Rielt — используют Atlas для привязки к локациям;
- наполнение:
  - смесь редакционного и демо-контента.

## Цели v1.0 (перезапуск проекта)

1. **Структура данных**
   - доопределить модель District/Area;
   - стандартизировать PlaceCategory и Tag под UX фильтров Atlas/Guru.

2. **Полноценный admin/UGC workflow**
   - чёткий pipeline:
     - `draft → pending_review → published → archived`;
   - поддержка UGC от PRO-пользователей;
   - backoffice-инструменты модерации.

3. **Поиск и фильтрация**
   - устойчивый nearby-поиск с PostGIS;
   - автодополнение по названиям мест и городов;
   - фильтры по тегам, типам, RF-флагу.

4. **Интеграция с RF**
   - единая модель привязки RF-партнёров к Atlas местам;
   - RF-метки в данных Atlas;
   - сценарии обновления при изменении статуса партнёров.

5. **Интеграция с другими сервисами**
   - финализировать контракты для Pulse, Rielt, Quest, Media;
   - сформировать stable API для внутренней экосистемы.

## Дальнейшее развитие (v2+)

1. **Глобальная карта и расширение географии**
   - добавление новых регионов/стран;
   - richer данные по странам/городам (факты, статистика).

2. **Внешний публичный API**
   - возможность использования Atlas третьими лицами (боты, партнёрские сайты);
   - throttling, API-ключи, тарифы.

3. **AI-обогащение данных**
   - авто-генерация описаний мест на основе скрапинга/AI;
   - авто-классификация тегов по текстам и отзывам;
   - рекомендации по добавлению новых мест (где «дыры» в карте).

4. **Исторический слой**
   - хранение изменений мест (например, «открыто», «закрыто», «переехало»);
   - связь с историей событий (Pulse) и контента (Blog).

5. **Глубокая интеграция с Recommendation Service**
   - персонализированные подборки мест в Atlas:
     - «места, которые вам могут понравиться»;
   - сигналы из Reactions/Quest/Space для рейтингования.

6. **Performance & Reliability**
   - вынесение поиска в отдельный Search Service;
   - отдельные read-replicas БД для Atlas;
   - улучшенный кеш по гео-тайлам для Guru.

## Риски и допущения

- Качество UX Guru и других модулей сильно зависит от полноты и качества данных Atlas.
- Наполнение каталога мест требует:
  - участия редакции;
  - инструментов для PRO и сообщества;
  - (в будущем) AI-дополнения.
- Atlas — база для Siberia Life, поэтому дальнейший roadmap должен учитывать повторное использование модели в других проектах (NSK/Siberia, Go2Asia, и т.д.).
