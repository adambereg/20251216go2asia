# Atlas Service — Wiki-энциклопедия локаций

## Назначение

Atlas Service — это центральный справочник геоданных для всей экосистемы Go2Asia.  
Он хранит информацию о:

- странах;
- регионах / городах / районах;
- конкретных местах (POI: достопримечательности, районы, пляжи, кафе, рестораны и т.п.).

Atlas используется:

- как **база знаний для модуля Atlas Asia** (страны → города → места);
- как **единый источник истины по локациям** для других сервисов (Pulse, Rielt, RF, Guru, Blog и др.). :contentReference[oaicite:1]{index=1}  

## Зона ответственности

Atlas Service отвечает за:

- хранение и выдачу структурированных геоданных;
- иерархию: страна → город → район → место;
- базовые атрибуты мест (название, координаты, тип, теги, описания, медиа);
- связь с RF-партнёрами через флаг `rf_badge` и `rf_place_id`;
- API для чтения и частично для записи/модерации данных;
- валидацию и модерацию изменений (editor + UGC модель).

## Что входит в Atlas

- **Countries** — список стран ЮВА и других регионов, в которых работает Go2Asia;
- **Cities** — города внутри стран (с привязкой к стране, базовыми координатами, slug);
- **Districts / Areas** — районы/зоны внутри города, если нужны для фильтрации (например, районы Бангкока);
- **Places** — любые «точки интереса» (POI):
  - урбанистические и туристические места;
  - инфраструктура (станции, районы, пляжи);
  - заведения бизнес-партнёров (через связь с RF).

## Что НЕ входит в Atlas

Atlas **не отвечает** за:

- события (это Pulse Service);
- объявления жилья, цены и доступность (это Rielt Service);
- ваучеры, скидки и акционные предложения (это Voucher / RF Service);
- пользовательские посты и отзывы (это Content/Reactions/Space, но они могут ссылаться на `place_id`);
- токеномику и балльную систему (это Token/Points/NFT).

Atlas хранит **только справочные данные о месте** и минимальный набор рейтинговых/агрегированных атрибутов (например, средний рейтинг, количество отзывов — через интеграцию с Reactions/RF).

## Основные потребители Atlas

- **Guru Service** — для поиска «мест рядом» и определения города по координатам; :contentReference[oaicite:2]{index=2}  
- **Pulse Service** — для привязки событий к месту проведения (`place_id`);
- **Rielt Service** — для привязки объявлений к городу/району, геокодинг адреса;
- **RF Service** — для связи партнёрских заведений с объектами Atlas (`atlas_place_id`, `rf_badge`);
- **Media Service (Blog)** — привязка статей к городам/местам;
- **Quest Service** — выбор мест как шагов квестов;
- **Recommendation / AI-модули** — как источник контекста о локациях.

## Влияние на UX

Atlas — фундаментальная часть UX:

- любая карта, список мест, фильтрация по локациям — работают поверх Atlas;
- единая система `country_id` / `city_id` / `place_id` позволяет консистентно связывать сущности по всей экосистеме;
- изменение объекта в Atlas (например, название места) **автоматически отражается** во всех модулях, которые используют `place_id`. :contentReference[oaicite:3]{index=3}
