# Rielt Service — Roadmap

## Этап 1 — MVP

- Объявления:
  - PropertyObject + Listing (аренда, базовые поля).
  - CRUD для авторов.
  - Статусы: `draft`, `published`, `archived`.
- Каталог:
  - поиск по стране/городу/типу/цене/комнатам/удобствам.
  - просмотр карточек объявлений.
- Заявки:
  - создание `TenantRequest`,
  - просмотр “моих заявок” и заявок по своим объявлениям.
- Избранное:
  - добавить/убрать объявление в избранное.

## Этап 2 — Партнёрские объекты и статистика

- Флаг `is_partner_object` и интеграция с RF Service.
- Статистика просмотров и заявок.
- Улучшенный фильтр каталога (районы, карта).

## Этап 3 — Улучшенный поиск и интеграция с контентом

- Текстовый поиск по описанию/заголовку.
- Поток отзывов/опыта через Content Service.
- Вывод средней оценки (через Reactions/Content).

## Этап 4 — Продвинутая воронка и бронирование

- Дополнительные статусы:
  - `booked`, `rented`.
- Базовое API взаимодействия с Booking/Payment Service (если будет выделен).
- Расширенная аналитика по конверсии (просмотр → заявка → сделка).

## Этап 5 — Рекомендации и AI

- Рекомендации объявлений пользователю (на основе его профиля, запросов).
- Интеграция с AI/ML для ранжирования и подсказок (например, “похожие объекты”).
