# Rielt Service — Обзор

## Назначение

Rielt Service — микросервис для модуля **Rielt.Market Asia** (рынок долгосрочной/краткосрочной аренды жилья в странах ЮВА).

Он отвечает за:

- хранение и управление объектами недвижимости и объявлениями;
- приём заявок арендаторов (interest / запрос на просмотр / бронирование);
- базовую воронку сделок (отклик → просмотр → бронь → сделка);
- связь объявлений с:
  - городами/локациями (Atlas),
  - партнёрскими заведениями/владельцами (RF / User Service),
  - пользовательским контентом (отзывы через Content/Reactions).

**Важно:**  
Rielt Service — source of truth по **объявлениям, объектам и заявкам**.  
UGC вокруг недвижимости (отзывы, обсуждения) живёт в **Content Service**, лайки/оценки — в **Reactions Service**.

## Основные сценарии

- Размещение объявлений:
  - владелец/арендодатель/агент создаёт объявление;
  - указывает параметры: тип объекта, количество спален, цена, период аренды, депозит, доступность, фото (через Media Service).
- Каталог для арендатора:
  - поиск жилья по городу/району/карте;
  - фильтры: цена, комнаты, срок аренды (короткий/долгий), удобства.
- Заявки/отклики:
  - пользователь оставляет запрос “интересуюсь этим объектом”;
  - арендодатель/агент видит список входящих запросов и связывается с арендатором.
- Статусы объявления:
  - `draft`, `published`, `paused` (временно скрыто), `archived`, `booked`, `rented`.
- В будущем:
  - интеграция с платёжными/бронированиями (пока вне Rielt / отдельный Booking / Payment Service).

## Роль в экосистеме

- **Atlas Asia**:
  - может отображать объекты/объявления на карте вместе с местами.
- **Space Asia / Content Service**:
  - пользователи пишут отзывы об объектах/районах;
  - делятся постами “как мы нашли жильё”.
- **RF Service**:
  - статус “партнёрский объект” (Russian Friendly landlord/агент);
  - дополнительные флаги, бонусы в Points/NFT.
- **Guru Service**:
  - PRO-гиды могут быть связаны как локальные агенты/консультанты по району.

Rielt Service должен быть готов к возможному росту до большого числа объявлений и запросов, но для MVP можно начать с простого решения.
