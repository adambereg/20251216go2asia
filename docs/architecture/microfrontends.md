# Microfrontends — Архитектура модульного фронтенда Go2Asia

---

## 1. Цели микрофронтендов

- Разделить ответственность между модулями (Atlas, Pulse, Blog и т.д.).
- Позволить развивать модули независимо.
- Сохранить единый UX и визуальный стиль.

---

## 2. Модель интеграции

Выбран подход **«single shell + modular routes»**:

- Одна PWA-оболочка (shell), внутри которой живут все модули.
- Каждый модуль:
  - имеет собственный набор маршрутов,
  - собственные контейнеры/страницы,
  - свои slice-ы состояния (если нужно),
  - но использует общую дизайн-систему.

---

## 3. Ответственность модулей

Пример:

- **Atlas**: отображение стран, городов, мест, гайдов.
- **Pulse**: события и календарь.
- **Blog**: статьи, лонгриды, обзоры.
- **Guru**: места, события, жильё и люди (PRO) "рядом со мной".
- **RF**: каталоги партнёров и ваучеры.
- **Rielt**: недвижимость.
- **Space**: социальная часть, лента, личные публикации.
- **Quest**: задания, маршруты, игровой слой.
- **Connect**: рефералы, амбассадоры.

---

## 4. Общие элементы

- Общий layout (header/footer/sidebar).
- Общая навигация по модулям.
- Общие UI-компоненты (кнопки, карточки, формы).
- Общий API-клиент и слой авторизации.

---

## 5. Переход между модулями

Переходы между модулями должны быть:
- лёгкими (без полной перезагрузки приложения),
- визуально понятными (акцент на том, в каком модуле пользователь сейчас).

---

## 6. Добавление нового модуля

Шаги:
1. Определить роль модуль в общей карте модулей (`go2asia_modules_map.md`).
2. Создать фронтенд-каталог модуля (страницы, контейнеры, компоненты).
3. Подключить маршруты в PWA shell.
4. Настроить доступ к API соответствующего backend-сервиса.
5. Добавить модуль в навигацию.
