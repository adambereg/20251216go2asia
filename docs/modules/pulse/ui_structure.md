# Pulse Asia — UI Structure

Версия 1.0  
Стартовая страница модуля Pulse Asia — **месячный календарь событий**.

---

## 1. Общие принципы UI

- Единый PWA-shell Go2Asia (верхняя навигация: Atlas, Pulse, Blog, Space).
- Фирменный **hero-баннер Pulse Asia** с названием модуля и описанием:
  > «События и мероприятия в Юго-Восточной Азии».
- Упор на **календарный формат** (Month / Week / Day / List).
- Сильные фильтры по:
  - стране,
  - городу,
  - месту (atlas_place),
  - типу события / категории,
  - режиму просмотра (онлайн/офлайн — позже).
- Интеграция с Atlas Asia:
  - карточки стран / городов ведут на список событий в Pulse,
  - события привязаны к `atlas_place_id`, `city_id`, `country_id`.
- Интеграция со Space Asia:
  - блок **«Отчёты сообщества»** под описанием события,
  - репост события в личную ленту и тематические группы (через Reactions Service).

---

## 2. Главная страница модуля (Month View)

**URL:** `/pulse` или `/pulse/calendar`

### 2.1. Верхний блок

1. **Hero-баннер**
   - Логотип/иконка Pulse Asia.
   - Заголовок `Pulse Asia`.
   - Подзаголовок: короткое описание миссии модуля.
2. **Панель управления календарём**
   - Кнопки вида:
     - `Месяц` (по умолчанию),
     - `Неделя`,
     - `День`,
     - `Список`.
   - Навигация по датам:
     - кнопка «Сегодня»,
     - стрелки «◀ / ▶»,
     - текущий диапазон дат: `ноябрь 2025 г.`.
3. **Глобальный поиск событий**
   - input с placeholder:
     > «Поиск по названию, организатору, темам…»

### 2.2. Фильтры

Строка «чипов» под поиском:

- **Страна** (`Все страны`, выбор конкретной страны).
- **Город** (динамически после выбора страны).
- **Место** (выбор из связанных atlas_places).
- **Тип события** (`Все`, `Онлайн`, `Офлайн` — позже).
- **Категории** (как набор тегов):
  - Еда,
  - Культура,
  - Музыка,
  - Спорт,
  - IT,
  - Сообщество,
  - Семья,
  - Ночная жизнь и т.п.

Фильтры работают совместно и мгновенно обновляют календарь и список.

### 2.3. Календарная сетка (Month)

Блок в центре страницы:

- Сетка 7×5/6 с днями месяца.
- В каждой ячейке:
  - дата (число),
  - цветные точки / мини-марки событий (по категориям).
- Клик по дню:
  - подсвечивает его,
  - в нижней части страницы открывает список событий этого дня (scroll-якорь).

### 2.4. Лента событий под календарём

Блок **«События: [выбранная дата/диапазон]»**:

- Список карточек событий по выбранному дню/фильтрам.
- Карточка события (компактный вид):
  - бейдж «Проверено» (если подтвержден модерацией),
  - название события (кликабельно),
  - короткое описание,
  - дата/время,
  - страна / город / место,
  - организатор,
  - индикаторы:
    - количество участников,
    - статус доступности/цены (например, «бесплатно», «от 600 THB»).

## 3. Неделя / День / Список

### 3.1. Week View

**URL:** `/pulse/week` (или параметр `view=week`)

- Верхняя панель — та же (поиск + фильтры).
- В центре:
  - горизонтальный календарь на 7 дней,
  - для каждого дня краткий список событий.
- Внизу — вертикальный список событий текущей недели.

### 3.2. Day View

**URL:** `/pulse/day` или `/pulse/[date]`

- Большой блок с выбранной датой:
  - день недели, дата,
  - количество событий.
- Под ним — **список событий дня**:
  - более крупные карточки, чем в Month/Week.
  - упор на время проведения, место и категорию.

### 3.3. List View

**URL:** `/pulse/list`

- Чистый список событий без визуального календаря.
- Сгруппировано по дням:
  - `Понедельник, 3 ноября`,
  - под заголовком — список событий.
- Удобен для пользователей, предпочитающих простую ленту.

## 4. Страница события (Event Detail)

**URL:** `/pulse/events/[event_slug_or_id]`

### 4.1. Hero-блок

- Большой баннер-фото события.
- Над фото — breadcrumb:
  - `Pulse Asia / [категория] / [название события]`.
- Под фото:
  - Название события (h1).
  - Подзаголовки/теги:
    - «Проверено» (если модерация),
    - категории (еда, культура, бизнес и т.д.),
    - длительность,
    - формат (воркшоп, фестиваль, митап).
  - Панель действий:
    - `RSVP` / «Пойду» (в будущем),
    - `Сохранить` (bookmark → Reactions Service),
    - `Репост` (в Space Asia),
    - контекстное меню «…» (пожаловаться, поделиться ссылкой).

### 4.2. Основной контентный блок

Структура центральной колонки:

1. **Блок «О событии»**
   - Основное описание.
   - Формат, ключевые активности, для кого подойдёт.
2. **Блок «Программа / Формат»** (опционально).
3. **Блок «Практическая информация»**
   - Что взять с собой, дресс-код, ограничения.
4. **Блок «Связанные места»**
   - Список 2–4 мест из Atlas, связанных с событием.
5. **Блок «Связанные темы»**
   - Темы Atlas (например, «Бизнес и работа», «Еда и культура»).

### 4.3. Правая колонка (инфо-карточки)

Правая часть страницы (стек карточек):

1. **Карточка даты и времени**
   - День недели, дата, время начала/окончания.
2. **Локация**
   - Город, страна.
   - Название места (с переходом в Atlas Place).
   - Кнопка «Смотреть на карте».
3. **Организатор**
   - Название,
   - ссылка на профиль/страницу в Space (позже).
4. **Участие и цена**
   - Кол-во мест,
   - текущий статус (свободно / почти заполнено / sold out),
   - цена (валюта локальная).
5. **Статус участия пользователя** (MVP+)
   - «Вы записаны / не записаны».

### 4.4. Блок «Отчёты сообщества»

- Заголовок: **«Отчёты сообщества»**.
- Под ним:
  - если отчётов нет:
    - пустое состояние: «Пока нет отчётов о событии. После мероприятия здесь появятся фото и впечатления участников.»
  - если отчёты есть:
    - сетка/лента карточек постов из Space Asia, связанных через Reactions/Quest/Report.
- Каждая карточка отчёта:
  - автор (аватар + имя + статус VIP/PRO),
  - короткий текст,
  - превью фото,
  - лайки/репосты (через Reactions Service),
  - ссылка в Space Asia.

## 5. Форма создания / редактирования события (MVP+)

**URL:** `/pulse/events/new` или `/pulse/events/[id]/edit`  
(Доступно только редакции и, возможно, PRO-организаторам в будущем.)

Основные поля:

- Название.
- Краткое описание (summary).
- Полное описание.
- Дата/время начала и окончания.
- Страна, город, место (выбор из Atlas).
- Категории (чипы).
- Формат (онлайн/офлайн).
- Цена и валюта.
- Ведущий / организатор.
- Hero-изображение.
- Настройки видимости:
  - опубликовано / черновик,
  - публичное / только для группы (в будущем).

UI следует общему формату форм Go2Asia: многошаговый мастер или одна страница с секциями.

## 6. Интеграции Pulse Asia в другие модули

### 6.1. Виджеты в Atlas

- На странице страны/города Atlas:
  - блок **«Ближайшие события»** (3–5 карточек, ссылка «Смотреть все в Pulse»).
- На странице места Atlas:
  - блок **«События в этом месте»**.

### 6.2. Интеграция со Space

- Репост событий в:
  - личную ленту,
  - тематические группы.
- Автоматический пост-отчёт (Phase 2+):
  - после участия и публикации отзывов/фото.

## 7. Адаптивность

- **Desktop (основной режим)**:
  - широкий календарь,
  - двухколоночный layout на странице события.
- **Tablet**:
  - компактная сетка календаря,
  - блоки события перестраиваются в одну колонку, правая колонка уходит вниз.
- **Mobile**:
  - по умолчанию — **Day/List view**:
    - месячный календарь свернут в верхнюю панель,
    - основной фокус — список событий для выбранной даты.
  - на странице события:
    - сначала hero-картинка,
    - далее — ключевая информация и описание,
    - в самом низу — отчёты сообщества.

---

## 8. Состояния и пустые экраны

- Нет событий по выбранным фильтрам:
  - экран с текстом «По выбранным фильтрам событий не найдено» и кнопкой «Сбросить фильтры».
- Событие завершилось:
  - бейдж «Событие прошло»,
  - основной CTA меняется с RSVP на «Смотреть отчёты».
- Ошибки загрузки:
  - универсальный компонент `EmptyState` по дизайн-системе Go2Asia.
