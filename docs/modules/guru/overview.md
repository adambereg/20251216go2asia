# Guru Asia — обзор модуля

## Роль в экосистеме Go2Asia

**Guru Asia** — это “живой” геогид по ближайшим объектам:
- местам (из Atlas Asia),
- событиям (из Pulse Asia),
- жилью (из Rielt.Market Asia),
- бизнес-партнёрам Russian Friendly,
- квестам Quest Asia,
- персональным проводникам и PRO-кьюраторам.

Guru — это не ещё один каталог, а **слой “что вокруг меня прямо сейчас”**.
Пользователь заходит в Guru, видит карту с точками и список карточек справа, выбирает лучшее, что есть поблизости.

Основные характеристики:

- **Геоцентричный интерфейс.** Карта (OpenStreetMap) + список по текущей локации.
- **Единый поток объектов разных типов.** Места, события, жильё, сервисы, квесты и проводники в одном списке.
- **Фильтры по интересам.** Еда, кофе, работа, тусовки, детские активности, экскурсии, Russian Friendly и др.
- **Глубокая интеграция с модулями.** Переход в карточку места/события/жилья/квеста в соответствующем модуле.
- **Работа как PWA.** Guru — идеален “в поле”: с телефона, на ходу, рядом с реальными объектами.

## Целевые сценарии

1. **Турист только что вышел из отеля.**
   - Открывает Guru → видит карту и список ближайших кафе, достопримечательностей и событий.
   - Фильтрует: “Еда и кофе” + “Russian Friendly”.
   - Выбирает место, открывает подробности в Atlas / RF.

2. **Digital nomad ищет, где поработать.**
   - Включает фильтр “Коворкинги и кафе для работы”.
   - Guru показывает ближайшие места с хорошим рейтингом, Wi-Fi и розетками.
   - Переход в Atlas для подробностей или в Rielt для долгосрочного жилья рядом.

3. **Житель/экспат планирует вечер.**
   - Выбирает период “сегодня вечером”, категории “музыка”, “вечеринки”.
   - Видит ближайшие мероприятия из Pulse Asia, сравнивает по цене и формату.
   - Сохраняет понравившиеся, открывает квест/событие для покупки ваучера или участия.

4. **Пользователь ищет Russian Friendly-сервисы поблизости.**
   - Включает фильтр “Russian Friendly”.
   - Guru показывает партнёрские заведения, услуги, русскоязычных гидов и организаторов.

5. **Игровой/квестовый сценарий.**
   - Пользователь выполняет квест Quest Asia.
   - Guru показывает чекпоинты квеста на карте, ближайшие связанные объекты, партнёрские места с ваучерами.

## Пользовательские роли

- **Гость / неавторизованный.**
  - Может видеть карту и список ближайших объектов.
  - Ограниченные фильтры, без персонализации и сохранений.

- **Spacer (базовый пользователь).**
  - Расширенные фильтры.
  - Сохранение мест/событий в избранное (через Reactions Service).
  - Переход в другие модули и реферальные сценарии.

- **VIP-spacer.**
  - Личная подборка “для вас”.
  - Доступ к части VIP-активностей и квестов.
  - Уведомления о релевантных событиях/местах (через Notification Service).

- **PRO-spacer и кураторы.**
  - Видят те же интерфейсы, но:
    - могут включать специальные QA-режимы (проверка карточек на местности),
    - использовать Guru как рабочий инструмент для инспекции Russian Friendly / Rielt / квестов.

- **Администраторы.**
  - Специальные режимы диагностики:
    - тепловые карты активности,
    - просмотр проблемных данных (ошибочные координаты, устаревшие объекты),
    - навигация к объектам для проверки.

## Стартовая страница модуля

Стартовый экран (на десктопе и планшетах):

- слева — карта (OpenStreetMap) с текущей локацией и кругом радиуса поиска;
- справа — вертикальный список карточек объектов: место / событие / жильё / сервис / квест;
- сверху над картой и списком — панель фильтров и быстрой смены радиуса;
- на мобильном — переключение “Карта / Список”.

## Интеграции модуля

Guru — фронтенд-модуль над **Guru Service** и рядом бэкенд-микросервисов:

- **Atlas Service** — места, города, страны.
- **Pulse Service** — события и мероприятия.
- **Rielt Service** — объявления жилья.
- **Voucher / RF Service** — партнёрские заведения и ваучеры.
- **Quest Service** — квесты и чекпоинты.
- **Reactions Service** — лайки, рейтинги, сохранённое.
- **Connect / Points / NFT** — награды за посещения мест, участие в квестах и активности.
- **Notification Service** — пуши “рядом с вами интересное событие”.

Guru не хранит “контент” сам по себе — он собирает данные из этих сервисов по координатам и превращает их в удобный интерфейс “здесь и сейчас”.
