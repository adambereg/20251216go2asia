# Guru Asia — Roadmap

Версия: 1.0  
Статус: черновик, согласовано как ориентир для разработки MVP и следующих фаз.

## Фаза 0 — Прототип и техническая база

**Цель:** быстро проверить UX‑концепцию «карта + список рядом» и интеграцию с OpenStreetMap.

### Задачи

1. Статический прототип:
   - Верстка layout’а *карта слева, список справа*.
   - Тестовые данные (mock JSON) для 10–20 объектов.
2. Интеграция OSM/Leaflet:
   - Показ карты, маркеров и радиуса.
   - Связка hover/клик маркера ↔ подсветка карточки.
3. Базовые фильтры:
   - Фильтр по типу (место/событие).
   - Фильтр по радиусу (фиксированное значение).
4. Подготовка SDK:
   - Клиент Atlas/Pulse для будущей интеграции.
5. Базовая аналитика:
   - События просмотра и кликов (Logging & Analytics Service).

**Результат:** понятный интерфейс Guru, работающий на мок‑данных, пригодный для юзер‑тестов.

## Фаза 1 — MVP Guru Asia (интеграция с Atlas/Pulse/RF)

**Цель:** выпустить рабочий Guru на продуктив с core‑функциями «что рядом сейчас».

### Основные функции

1. **Получение реальных данных:**
   - Интеграция с Atlas Service (места, города).
   - Интеграция с Pulse Service (события).
   - Интеграция с RF Service / Voucher Service (партнёры и наличие ваучеров).

2. **Агрегированный Guru Feed:**
   - Эндпоинт `/api/guru/feed` на API‑шлюзе.
   - Нормализация разных сущностей в `GuruFeedItem`.
   - Сортировка по расстоянию/времени/рейтингу.

3. **Продвинутые фильтры:**
   - Типы: места, события, партнёры RF.
   - Категории: еда, бар, коворкинг, экскурсии, музыка и т.д.
   - Открыто сейчас / только Russian Friendly / с ваучерами.
   - Радиус: 500 м – 5 км.

4. **Связка с другими модулями:**
   - Переходы на Atlas/Pulse/RF/Quest/Rielt по клику «Открыть».
   - Передача UTM/аналитики в Logging & Analytics.

5. **Состояние сессии:**
   - Хранение настроек (фильтры, радиус, вид) в URL и localStorage.
   - Простая история «вы недавно смотрели».

6. **Награды и реакции:**
   - Лайки/сохранения через Reactions Service.
   - Базовые триггеры в Connect Service (Points за активности в Guru — см. токеномику).

**Критерии готовности MVP:**  
Пользователь заходит в Guru, видит карту и список мест/событий вокруг, может применить фильтры, открыть карточку в других модулях и сохранить понравившиеся варианты.

## Фаза 2 — AI‑подсказки и персонализация

**Цель:** превратить Guru из «списка мест» в умного ассистента, предлагающего сценарии.

### Основные функции

1. **AI‑подбор сценариев:**
   - Эндпоинт `/api/guru/ai/suggestions` (через AI Gateway).
   - Подбор наборов объектов под контекст (пара, семья, работа, ночь и т.п.).
   - Отображение сценариев блоком над лентой.

2. **Учёт ролей и уровня участия:**
   - Различные сценарии для guest/spacer/VIP/PRO.
   - Приоритет RF‑партнёров и квестов с ваучерами для VIP/PRO.

3. **Персонализация по истории:**
   - Использование истории просмотренных и сохранённых объектов.
   - Базовый рекомендательный слой на стороне AI Gateway (без сложного ML на старте).

4. **Интеграция с Connect/Points/NFT:**
   - Награды за выполнение AI‑рекомендаций (например, посетил 3 объекта из плана).
   - Потенциальные NFT‑бейджи за «экспедицию» по городу.

5. **Улучшенная аналитика:**
   - Трекинг, какие подсказки принимаются и какие сценарии используются.
   - Фидбек‑петля для улучшения промтов и логики AI.

## Фаза 3 — Глубокая интеграция с квестами и ваучерами

**Цель:** сделать Guru центральной точкой входа в игровые и экономические механики Go2Asia.

### Основные функции

1. **Квесты рядом:**
   - Выделенный слой для квестов Quest Asia в радиусе.
   - Отображение прогресса по квесту прямо в интерфейсе Guru.
   - Быстрый старт/продолжение квеста «из карты».

2. **Ваучеры рядом:**
   - Места, где можно использовать текущие ваучеры пользователя (из Voucher Service).
   - Подсветка карточек с доступными премиум‑ваучерами.

3. **Сценарии “квест + ваучер”:**
   - AI‑подсказки, комбинирующие квесты, визиты к партнёрам RF и использование ваучеров.

4. **Интеграция с Space и репостами:**
   - Лёгкий репост маршрута/сценария в Space.
   - Возможность PRO оформлять через Guru тематические «прогулки», которые затем становятся квестами/гид‑лонгридами в Atlas/Quest.

## Фаза 4 — Расширенные режимы и монетизация

**Цель:** превратить Guru в ключевой «повседневный» инструмент для резидентов и путешественников.

### Основные направления

1. **Режим “жизнь в городе”:**
   - Отдельные пресеты фильтров для long‑stay резидентов (коворкинги, сервисы, регулярные события).

2. **Интеграция с Rielt/Accommodation:**
   - Режим “исследовать район перед арендой” — связка с объявлениями жилья.

3. **Партнёрские витрины:**
   - Специальные тематические виджеты партнёров (например, фестиваль, региональные программы).

4. **Более сложная персонализация/ML:**
   - Сегментация пользователей.
   - Тонкая настройка рекомендаций с использованием накопленных логов.

---

## Приоритеты

- **Сейчас (MVP):** Фаза 1 (core‑функции Guru с интеграцией Atlas/Pulse/RF + базовые реакции).  
- **Дальше:** Фаза 2 (AI‑подсказки) и выборочная интеграция элементов Фазы 3 (квесты/ваучеры), если к этому моменту будут готовы Quest/Voucher/Connect микросервисы.

Guru Asia — один из ключевых модулей, связывающих контент, события, партнёров и токеномику Go2Asia в реальном геоконтексте. Именно здесь пользователи «чувствуют» город и платформу в повседневной жизни.
