# Blog Asia — UI Structure

Документ описывает ключевые экраны и блоки UI для модуля Blog Asia в PWA‑оболочке Go2Asia.
Цель — дать Cursor/дизайнерам и разработчикам стабильный каркас интерфейса, который уже частично реализован в прототипах.

---

## 1. Главная страница Blog Asia

**URL:** `/blog`

**Цель:** Быстро показать живую картину контента: главные редакционные материалы, выбор сообщества, рубрики, события недели, блоки «Жить здесь» и партнёрские истории.

### 1.1. Hero‑блок

- Большой баннер с:
  - заголовком топ‑статьи,
  - коротким описанием,
  - метками (страна/город, рубрика, тип материала: Editorial/UGC/Partner),
  - временем чтения,
  - бейджем «Выбор редакции» или «Новый».
- Фон — обложка статьи во всю ширину центра контейнера (с единым aspect‑ratio, как в Atlas/Pulse).

### 1.2. Навигация по рубрикам и темам

Под hero‑блоком:

- горизонтальный список основных рубрик (chips):
  - Путешествия
  - Работа и доход
  - Жить здесь
  - Гайды и подборки
  - Бизнес и партнёры
  - Практика и документы
- дополнительный фильтр по темам (tags-хабы): «Digital nomads», «Семья и дети», «Переезд», «Финансы» и т.п.
- строка поиска по статьям (поиск по заголовкам/темам/авторам).

### 1.3. Блок «Главные сегодня»

Секция карточек 3–6 статей:

- каждая карточка:
  - обложка,
  - заголовок,
  - короткий лид,
  - страна/город (иконки Atlas),
  - тип материала (EDITORIAL/UGC/Partner),
  - время чтения,
  - минимальная статистика (просмотры, реакции).

Логика наполнения:
- редакция вручную фиксирует `featured` статьи,
- автоматический fallback — самые популярные за 7 дней.

### 1.4. Блок «Выбор сообщества»

- 2–4 карточки статей, выросших из UGC (Space Asia).
- маркировка «Выбор сообщества».
- сортировка по количеству репостов/реакций за последние N дней.

### 1.5. Блок «Рубрики»

- Сетка рубричных карточек:
  - Путешествия,
  - Работа и доход,
  - Жить здесь,
  - Бизнес и партнёры и т.п.
- Каждая карточка ведёт на рубричную страницу (`/blog/category/{slug}`).

### 1.6. Блок «События недели» (интеграция с Pulse Asia)

- 1–3 статьи или мини‑подборки, связанные с событиями:
  - «Фестиваль огней в Чиангмае»,
  - «Неделя стартапов в Хошимине».
- Карточки могут содержать бейдж «Связано с событиями» и ссылку «Смотреть события в Pulse».

### 1.7. Блок «Жить здесь»

- 3–4 материала о долгосрочной жизни в городах: районы, стоимость жизни, инфраструктура.
- Тесная интеграция с Atlas/Rielt для перехода к местам и жилью.

### 1.8. Блок «Партнёрские истории»

- Специальный ряд карточек для кейсов Russian Friendly и партнёрских статей.
- Чёткая маркировка «Partner Story» согласно style guide.

## 2. Страница статьи

**URL:** `/blog/{slug}`

### 2.1. Hero статьи

- крупная обложка во всю ширину контейнера,
- над заголовком:
  - хлебные крошки: `Blog → Рубрика → Город`,
  - тип материала (EDITORIAL/UGC/Partner);
- под заголовком:
  - автор (аватар, имя, роль — PRO/редакция),
  - дата публикации,
  - время чтения,
  - метки: страна, город, тип аудитории (Digital nomad, семья, лонгстей и т.д.).

### 2.2. Основной контент

- контентная колонка с:
  - подзаголовками H2/H3,
  - списками, цитатами, таблицами,
  - вставками карточек мест/событий/квартир,
  - CTA‑блоками (например, «Смотреть места в Atlas», «Смотреть жильё в Rielt.Market»).
- единый стиль текста с соблюдением принципов go2asia_style_guide.

### 2.3. Боковая панель / правый столбец (на десктопе)

- блок «Контекст статьи»:
  - страна,
  - город,
  - рубрика,
  - бюджет/уровень (если задано),
  - тип путешественника.
- блок «Места из статьи»:
  - вертикальный список карточек мест Atlas (мини‑карточки).
- блок «Рекомендуем почитать»:
  - статьи по теме / в этом городе / этого автора.

На мобильном эти блоки уходят вниз после контента.

### 2.4. Блок взаимодействий

Под статьёй:

- кнопки «Понравилось», «Сохранить», «Поделиться» (репост в Space, копирование ссылки).
- короткий счётчик реакций (подключён к Reactions Service).
- блок «Отчёты сообщества»:
  - показываются ссылки на UGC‑посты в Space, которые репостят или обсуждают эту статью (посты‑отчёты, личный опыт).

## 3. Рубричные страницы

**URL:** `/blog/category/{slug}`

Содержат:

- заголовок рубрики,
- описание и мини‑гид по рубрике,
- список последних и популярных статей в этой рубрике (с пагинацией),
- дополнительные фильтры: страна, город, теги.

---

## 4. Тематические хабы / серии

**URL:** `/blog/hub/{slug}` или `/blog/series/{slug}`

Структура:

- крупный hero‑баннер с названием серии (например, «Переезд во Вьетнам»),
- описание: для кого серия, какие темы раскрывает,
- навигация по шагам/подтемам (например, «Визы», «Жильё», «Работа», «Дети и школы»),
- список статей в рамках серии, сгруппированных по шагам.

---

## 5. Страница автора

**URL:** `/blog/author/{slug}`

- обложка/аватар автора,
- краткая биография, города/страны экспертизы,
- бейджи PRO/NFT‑достижений,
- фильтруемый список статей автора:
  - по рубрикам,
  - по странам,
  - по типам материалов (подборки, лонгриды, партнёрские).

---

## 6. Состояния и адаптивность

- Пустые состояния:
  - «По этому фильтру пока нет статей» — с подсказками изменить фильтры или посмотреть популярное.
- Скелетоны загрузки для карточек и контентных областей.
- Адаптив:
  - мобайл: упрощённая структура, вертикальный скролл, hero и контент на всю ширину, навигация по рубрикам и фильтрам в horizontal scroll.
  - планшет: двухколоночный макет.
  - десктоп: как в прототипах — широкая центральная колонка + боковая.

---

## 7. Интеграция с PWA‑shell

- Blog Asia встроен в общий PWA‑shell:
  - топ‑навигация Atlas / Pulse / Blog / Space.
- По клику на связанные сущности:
  - открываются страницы модулей Atlas, Pulse, Rielt, Quest в пределах того же shell (без перезагрузки).
- По взаимодействиям (лайк, сохранение, репост) — вызываются методы Reactions Service через общие хуки SDK.

Этот документ должен использоваться как эталон при доработке UI‑компонентов Blog Asia в Storybook и при генерации макетов в Cursor.
