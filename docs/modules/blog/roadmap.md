# Blog Asia — Roadmap

Черновой план развития модуля Blog Asia с учётом текущего состояния прототипов и общей стратегии Go2Asia.

---

## Этап 0 — Текущий прототип (есть сейчас)

- Реализована PWA‑оболочка и базовый UI:
  - главная страница Blog Asia с hero‑баннером и блоками карточек,
  - страница статьи с контентом,
  - базовая навигация между Atlas / Pulse / Blog / Space.
- Контент статичен / захардкожен, либо подаётся через временные мок‑API.
- Нет полноценной редакционной очереди, нет экономических наград, нет глубокой связи с Reactions/Connect.

Цель Этапа 0: показать живой прототип и проверить восприятие структуры.

---

## Этап 1 — MVP блог‑движка (под управлением Content Service)

**Цель:** сделать Blog Asia реально работающим редакционным модулем на основе общих паттернов Content Service.

Основные задачи:

1. **Базовая CRUD‑функциональность статей**
   - API для списка статей, просмотра одной статьи, рубрик, тегов.
   - Внутреннее API для создания/редактирования/публикации статей.
   - Подключение к общей БД (Neon + Drizzle).

2. **Рубрики и теги**
   - Настроить базовый набор рубрик и тегов.
   - Рубричные страницы и фильтры на главной.

3. **Связь с Atlas/Pulse**
   - Возможность привязать страну/город/места/события к статье.
   - Вывод связанных сущностей в UI статьи (блоки «Места из статьи», «События»).

4. **SEO‑минимум**
   - Поля meta_title, meta_description, og‑image.
   - Генерация человекочитаемых slug’ов.

5. **Интеграция с Reactions Service**
   - Лайки, сохранения, репосты (через Reactions API).
   - Агрегированная статистика просмотров/реакций.

Результат: Blog Asia можно использовать как основной медиа‑раздел, статьи подаются через единый API, редакция публикует материалы вручную через админку/backoffice.

---

## Этап 2 — Редакционная машина и UGC‑трубопровод

**Цель:** связать Space Asia (UGC) и Blog Asia (редакция) в единую систему.

Задачи:

1. **Editorial Queue**
   - Реализация сущности EditorialQueue.
   - Воронка: «пост в Space» → «кандидат в статью» → «черновик статьи» → «публикация».
   - Интерфейсы для редакторов/PRO‑кураторов: список кандидатов, статусы, комментарии.

2. **Авто‑рекомендации кандидатов**
   - На основе сигналов Reactions/Views:
     - большое число репостов,
     - обсуждения,
     - высокий интерес к теме.
   - Создание карточек EditorialQueue автоматически.

3. **Серии и тематические хабы**
   - Введение сущности Series/Hub.
   - Страницы серий (переезд, визы, работа, digital‑nomad и т.п.).
   - Привязка статей к сериям, отображение на главной.

4. **Интеграция с Connect Service**
   - Событие `blog.article.published` → награды автору/кураторам:
     - Points за публикацию/прочтения/репосты.
     - NFT‑бейджи для авторов/редакторов по достижениям (через NFT Service).

Результат: Blog Asia становится полноценной редакционной плитой для переработки UGC‑контента в качественные статьи и спецпроекты.

---

## Этап 3 — Персонализация и рекомендательная система

**Цель:** повышать вовлечённость за счёт персональных лент и умных блоков.

Задачи:

1. **Персонализированные рекомендации**
   - Учет:
     - стран/городов интереса пользователя,
     - его роли (турист, longstay, инвестор),
     - истории реакций (репосты, лайки, сохранения).
   - Блоки «Рекомендуем для вас» на главной и в PWA‑shell.

2. **Глубокая интеграция с Atlas / Pulse / Rielt**
   - На страницах стран/городов/мест — блоки статей по контексту.
   - На страницах событий и объектов недвижимости — связанные статьи.

3. **Тематические дайджесты**
   - Подготовка email‑ и push‑дайджестов на базе Blog Asia + Atlas + Pulse.
   - Управление подписками (через Notification Service).

---

## Этап 4 — Монетизация и партнёрские форматы

**Цель:** превратить Blog Asia в инструмент привлечения партнёров и монетизации.

Задачи:

1. **Партнёрские размещения**
   - Особые типы статей/кейсов для Russian Friendly и Rielt‑партнёров.
   - Метаданные по кампаниям, срокам, UTM.

2. **Продвинутые CTA‑блоки**
   - Вставки:
     - «Забронировать жильё в этом районе» (Rielt.Market),
     - «Посмотреть партнерские заведения рядом» (Russian Friendly),
     - «Пройти квест по теме статьи» (Quest Asia).

3. **Аналитика по вкладу статей**
   - Трекинг переходов из статей в Atlas/Pulse/Rielt.
   - Отчёты: какие статьи лучше всего конвертят в регистрации, подписки, бронирования.

---

## Этап 5 — AI‑ассистенты и генерация контента

**Цель:** использовать AI как инструмент поддержки редакции и авторов.

Идеи:

- AI‑подсказки для структуры статьи, заголовков, лидов.
- Авто‑генерация коротких вытяжек/summary для PWA‑карточек.
- Смарт‑рекомендации по связям статьи с местами, событиями и квестами (анализ контента + данные Atlas/Pulse).
- В будущем — полуавтоматическое создание гайд‑серий на основе баз знаний Atlas/Quest/RF.

---

Эта дорожная карта не является жёсткой спецификацией, но задаёт направление развития Blog Asia в составе единой экосистемы Go2Asia: от статичного медиа‑раздела к живому, связному и экономически значимому контентному ядру.
