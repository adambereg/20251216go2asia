# Quest Asia — UI Structure

## 1. Общие принципы

- Модуль визуально продолжает общий стиль Go2Asia:
  - ModuleHero вверху страницы (как в Atlas / Pulse / Blog),
  - светлый фон, карты и карточки с soft-радиусами,
  - акцентный фиолетовый градиент для Quest Asia.
- В интерфейсе везде подчёркиваются:
  - тип квеста (Маршрут / Контент / Событие / Онлайн / Офлайн),
  - длительность, город, сложность,
  - награды (Points + NFT).

Основные пользовательские роли для UI:
- **Traveler/Explorer** — видит и проходит квесты.
- **PRO / Business** — создаёт и управляет квестами.
- **Guest** — видит список квестов и детальную страницу, но не может начать квест без авторизации.

## 2. Страницы для путешественника / экспата

### 2.1. Список квестов — `/quest`

**Hero-блок (ModuleHero)**  
- Заголовок: `Quest Asia`
- Подзаголовок: «Проходите квесты, выполняйте миссии и получайте награды»
- Фон: фиолетовый градиент (как на скрине).

**Секция “Доступные квесты”**
- Заголовок: `Доступные квесты`
- (MVP) Без фильтров, просто сетка карточек.
- (Расширение) Панель фильтров:
  - Город (селект),
  - Тип квеста (Маршрут, Контент, Событие, ONLINE/OFFLINE),
  - Сложность (Легко, Средне, Сложно),
  - Длительность (чипы: до 60 / 90 / 120+ мин),
  - Теги (Ночь, RF, NFT и т.п.).

**Сетка карточек квестов (4–6 в ряд на десктопе, 2 на таблетах, 1 на мобиле)**  
Компонент: **QuestCard**

Структура карточки:

1. **Верхняя строка бейджей**
   - Слева: бейджи статуса/типа:
     - `RF` — если квест завязан на Russian Friendly партнёров,
     - `Ночь`, `Онлайн`, `Офлайн` — режим выполнения.
   - Справа: основной тип квеста:
     - `Маршрут`, `Контент`, `Событие`, `Чек-ин`.

2. **Обложка**
   - Фото (16:9 или близкий формат),
   - hover-эффект (лёгкий zoom/overlay).

3. **Блок заголовка**
   - Название квеста (1–2 строки),
   - Краткий текст-тизер (1 строка).

4. **Инфо-строка**
   - Иконка времени + длительность: `90 мин`, `120 мин` и т.п.
   - Иконка локации + город/район: `Бангкок`, `Пхукет`, `Чиангмай`.
   - Чип сложности: `Легко`, `Средне`, `Сложно`.

5. **Награды**
   - Иконка трофея + `300 очков`, `500 очков`, и т.п.
   - Если есть NFT — бейдж `NFT`.

Клик по карточке ведёт на детальную страницу: `/quest/:slug` или `/quest/:id`.

---

### 2.2. Детальная страница квеста — `/quest/{id}`

Разметка — **два столбца**: контент слева, награды и CTA справа.

#### 2.2.1. Hero-секция

- Большое изображение (обложка квеста) по всей ширине.
- На оверлее:
  - Бейджи (RF / Онлайн / Ночь / Маршрут и т.д.).
  - Название квеста.
  - Краткий подзаголовок / описание (1–2 строки).

#### 2.2.2. Левая колонка

**Блок “О квесте” (карточка)**  
Содержит:

- `Длительность` — с иконкой (напр. `90 мин`).
- `Лучшее время суток` — напр. `Вечер`.
- `Локация` — город и район: `Бангкок, Чайна-таун`.
- `Стартовая точка` — адрес + ссылка `Открыть в картах` (открывает Guru / внешние карты).
- `Рекомендуемая подготовка` — список (аппетит, вода, удобная обувь и т.п.).
- (Опционально) `Тип квеста`, `Онлайн/офлайн`, `Возрастные ограничения`.

**Блок “Маршрут” (card)**  

Заголовок: `Маршрут` / `Задачи`.

Внутри — вертикальный список шагов (**RouteStep**):

Каждый шаг содержит:
- Порядковый номер (`1`, `2`, `3`).
- Тип: `Задание`, `Чек-ин`, `Посетите место`, `Попробуйте блюдо` и т.п.
- Название шага:
  - напр. `Попробуйте Pad Thai`, `Манго с клейким рисом`, `Том Ям`.
- Описание:
  - короткое пояснение (что сделать, где, за что даются очки).
- Доп. инфо:
  - `Радиус` (напр. `100 м`, `200 м`),
  - `Время на выполнение` (если необходимо).

**Блок “Маршрут на карте”**

- Плашка с заглушкой `Карта будет здесь` (MVP),
- В будущем — интерактивная карта Guru с маркерами шагов.

---

#### 2.2.3. Правая колонка

**Блок “Награды” (QuestRewardsCard)**  

- `Очки` — крупным числом (напр. `300 очков`).
- `NFT-награды`:
  - список NFT бейджей (например, `Street Food Hero`, `Bangkok Explorer`),
  - карточки в виде чипов.

**CTA-блок**

- Основная кнопка: `Начать квест` (primary, фиолетовая).
- Под кнопкой:
  - текст: `Квест будет сохранён в разделе ваших активных квестов`,
  - (неавторизованному) — подсказка `Чтобы начать квест, авторизуйтесь`.

Возможные дополнительные элементы:
- Статус:
  - `Вы уже проходите этот квест`,
  - `Квест завершён, вы заработали 300 очков и NFT ...`.

---

### 2.3. Экран/режим прохождения квеста — `/quest/{id}/play` (или оверлей)

Формат: один активный шаг за раз + прогресс-бар сверху.

**Элементы:**

- Хлебные крошки: `Quest Asia → Название квеста`.
- Прогресс-бар:
  - визуально показывает `Шаг 1 из 5`, `%` выполнения.
- Карточка текущего шага:
  - номер и заголовок,
  - тип действия (иконка: камера / локация / вопрос),
  - подробное описание,
  - при `visit_place` — карта/плашка с расстоянием,
  - при `take_photo` — UI для загрузки фото,
  - при `quiz` — поле ввода ответа / варианты ответа.
- Кнопки:
  - `Отправить` / `Завершить шаг`,
  - `Пропустить` (если это дозволено),
  - `Назад к списку шагов`.

**После завершения квеста:**

Экран “Квест завершён”:
- Анимация/иллюстрация успеха,
- Суммарные награды (Points + NFT),
- Кнопки:
  - `Поделиться в Space Asia`,
  - `Найти новые квесты`.

## 3. Страницы для PRO / бизнес-партнёров

### 3.1. Дашборд квестов — `/quest/dashboard`

**Для PRO / Business:**

- Заголовок: `Мои квесты`
- Кнопка `Создать квест`.
- Таблица/сеткой:
  - Название,
  - Город,
  - Статус (Черновик / Опубликован / Архив),
  - Кол-во участников,
  - Средний рейтинг (если появится),
  - Дата создания / последнего изменения.
- Действия:
  - `Редактировать`,
  - `Статистика`,
  - `Архивировать`.

---

### 3.2. Конструктор квеста — `/quest/new` и `/quest/{id}/edit`

**Шаг 1. Общая информация**

- Название квеста,
- Краткое описание,
- Тип квеста (Маршрут / Контент / Событие / Онлайн / Офлайн),
- Город / район,
- Длительность,
- Сложность,
- Обложка (загрузка изображения).

**Шаг 2. Настройка маршрута и шагов**

- Список шагов (вертикальный):
  - drag & drop сортировка,
  - тип шага: `visit_place`, `take_photo`, `quiz`, `checkin`, `attend_event`, `buy_item`, `custom`,
  - поля в зависимости от типа:
    - Place (выбор места из Atlas/Guru),
    - Event (из Pulse),
    - координаты,
    - текст задания,
    - радиус и/or время,
    - тип проверки (фото, GPS, ответ).
- Кнопки: `Добавить шаг`, `Дублировать`, `Удалить`.

**Шаг 3. Награды**

- Поле `Points за прохождение`,
- Выбор NFT (из каталога NFT-модулей) или создание нового бейджа,
- (Опция) `Купон RF` — выбор из списка купонов бизнес-партнёра.

**Шаг 4. Предпросмотр и публикация**

- Превью страницы квеста (приблизительно как `/quest/{id}`),
- Переключатель `Черновик / Опубликован`,
- Кнопка `Сохранить` и `Опубликовать`.

---

## 4. Вспомогательные UI-компоненты

- **QuestCard** — карточка квеста в списке.
- **QuestHero** — верхняя секция с обложкой и заголовком.
- **QuestMetaRow** — строка с длительностью, локацией, сложностью.
- **QuestRewardsCard** — блок наград (Points + NFT).
- **RouteStep** — элемент списка шагов.
- **QuestProgressBar** — индикатор прогресса прохождения.
- **QuestFilterBar** — (будущее) панель фильтров на /quest.
- **QuestStatusBadge** — бейдж статуса/типа.

---

## 5. Состояния и адаптивность

- **Гостевой режим**:
  - Можно просматривать список и детали квеста,
  - При нажатии “Начать квест” открывается модальное окно/редирект на `Sign In`.
- **Нет квестов**:
  - Плашка EmptyState: “В этом городе пока нет квестов. Попробуйте выбрать другой город или вернитесь позже”.
- **Мобильная версия**:
  - Hero сжимается,
  - Сетка квестов → 1 карточка в ряд,
  - Правая колонка на детальной странице уходит вниз (сначала описание, затем награды и CTA).
