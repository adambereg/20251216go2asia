Руководство по визуальному языку и пользовательским интерфейсам экосистемы Go2Asia

Версия 1.0
________________________________________
1. Цели и назначение style guide
Этот документ описывает единый визуальный язык Go2Asia и правила построения интерфейсов для всех модулей платформы:
•	Atlas Asia
•	Pulse Asia
•	Blog Asia
•	Guru Asia
•	Russian Friendly
•	Quest Asia
•	Rielt Asia
•	Space Asia
•	Connect (Tokenomics)
•	PWA Shell (оболочка)
Цель — обеспечить полную стилистическую и функциональную согласованность всех микрофронтендов, независимо от того, генерируются они человеком, Cursor или Bolt.New.
Документ обязателен для всех агентов разработки.
________________________________________
2. Дизайн-философия Go2Asia
2.1. «Лёгкость и позитив Юго-Восточной Азии»
Визуальный стиль должен передавать атмосферу:
•	тёплых оттенков,
•	ярких акцентов,
•	чистых форм,
•	лёгкой «воздушности».
2.2. Профессиональная простота
Интерфейсы должны быть:
•	лёгкими для восприятия,
•	минималистичными,
•	построенными на картах, карточках и блоках,
•	без перегрузки деталями.
2.3. Фокус на удобстве путешественника
UI должен быть оптимизирован под:
•	быстрый доступ к информации,
•	контекст (геолокация),
•	сценарии «рядом со мной»,
•	мобильное использование.
2.4. Единый визуальный язык
Все модули используют один набор дизайн-токенов:
•	цвета,
•	шрифты,
•	размеры,
•	отступы,
•	компоненты,
•	анимации.

5. Расстояния и сетка
5.1. Базовый модуль сетки
4px grid
Стандартные отступы:
•	XS — 4px
•	S — 8px
•	M — 12px
•	L — 16px
•	XL — 24px
•	XXL — 32px
5.2. Карточные раскладки
•	радиус скругления: 12px
•	тень: минимальная, air-light shadow
•	фон фото: скругление 12px (обязательно единообразно!)
________________________________________
6. Компонентная система
Набор UI-компонентов, используемых всеми модулями.
6.1. Buttons
Тип кнопки	Использование
Primary	CTA, бронирование, купить
Secondary	переключатели
Ghost	фильтры
Danger	отмена, удалить
Все кнопки имеют:
•	border-radius: 12px
•	высота: 44px
•	padding: 0 16px
________________________________________
6.2. Cards
Используются в Atlas, Pulse, Blog, RF, Quest, Rielt.
Требования:
•	единая рамка фотографии,
•	аккуратная тень,
•	структурированный блок: фото → заголовок → подзаголовок → метаданные.
Карточки должны сохранять единый стиль при генерации Bolt.New.
________________________________________
6.3. Hero-блоки
Единое правило:
Все hero-блоки — без рамки вокруг фото.
Используются в:
•	Atlas (страна, город),
•	Pulse (событие),
•	Blog (статья),
•	Quest (квест),
•	Rielt (объявление).
________________________________________
6.4. Chips / Tags
Используются для:
•	фильтров,
•	категорий,
•	тегов в Blog,
•	маркеров в Atlas.
________________________________________
6.5. Navigation Patterns
Главная навигация PWA:
•	Bottom bar (мобильные)
•	Left sidebar (desktop)
•	Плавающий FAB для Guru

7. Правила UI для модулей
7.1. Atlas Asia (Обязательные правила)
•	карточки мест — одинаковые везде
•	фото — с рамкой
•	hero — без рамки
•	вкладки: Обзор / История / Культура / Локации
•	структура города — всегда одинаковая
•	структура места — всегда одинаковая
7.2. Pulse Asia
•	календарные представления: список, сетка, афиша
•	цветовая схема: оранжевый акцент
•	карточки событий: однотипные
7.3. Blog Asia
•	карточки статей имеют рамку вокруг фото
•	hero статьи — без рамки
7.4. Guru Asia
•	карта обязательна
•	карточки гуру-PRO
•	единая структура: места/события/PRO рядом
7.5. Space Asia
•	лента — карточки контента
•	реакционная модель без комментариев
•	репосты — отдельный тип карточек
7.6. Quest Asia
•	карточки квестов
•	прогресс-бары
•	чекпоинты
7.7. Rielt Asia
•	карточки объявлений
•	структура: фото → цена → параметры
7.8. RF Catalog
•	фото заведений — с рамкой
•	бейджи: Verified, RF, Promo, etc.

10. Доступность
•	контраст текста WCAG AA
•	шрифты не менее 14px
•	фокус-стейт для всех интерактивных элементов
•	aria-labels для иконок
•	адаптивность для устройств от 320px
________________________________________
11. Правила для AI-агентов (Cursor / Bolt.New)
Cursor обязателен к соблюдению:
•	всех токенов,
•	всех компонентных правил,
•	всех структур карточек,
•	единообразия hero-блоков.
Bolt.New обязан:
•	использовать дизайн-токены,
•	генерировать карточки строго по правилам,
•	соблюдать рамки вокруг фото в карточках,
•	не изменять цветовую схему модулей.
________________________________________
12. Итог
Этот Style Guide гарантирует:
-一致ность всех модулей,
•	удобство интерфейса,
•	скорость разработки,
•	совместимость FE/BE,

Go2Asia — единая визуальная экосистема, в которой UI всегда предсказуем, узнаваем и дружелюбен.
